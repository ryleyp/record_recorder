## LAME 3.100 (the only third-party dependency)

macOS frameworks can *decode* MP3 but cannot *encode* it (AVFoundation's
encoders cover AAC/ALAC only), so the app bundles the LAME MP3 encoder.

- **What:** LAME `libmp3lame` 3.100 — encoder library only; the decoder
  (mpglib) and command-line frontend are not included.
- **Where:** `VinylAlbumRecorder/ThirdParty/lame/` — vendored C source,
  compiled directly into the app by Xcode. Nothing to install: building the
  Xcode project builds LAME. End users never need Homebrew or any package
  manager.
- **Source:** `lame-3.100.tar.gz` from the official SourceForge release,
  SHA-256 `ddfe36cab873794038ae2c1210557ad34857a4b6bdc515785d1da9e175b1da1e`.
- **Local modifications:** none to any `.c`/`.h` source file.
  `config.h` was generated by running LAME's own
  `./configure --disable-frontend --disable-decoder` on macOS (arm64) and is
  vendored alongside the sources. `mpglib_interface.c` (decoder glue) is
  omitted. `HAVE_CONFIG_H=1` is defined in the Xcode build settings.
- **License:** GNU Lesser General Public License v2.1. The full text is
  vendored at `VinylAlbumRecorder/ThirdParty/lame/COPYING`.

### LGPL-2.1 compliance notes

LAME is statically compiled into the app binary. The LGPL's requirements are
met for this open-source project because:

1. The complete corresponding source of both the application and the LAME
   library is published in this repository, and
2. Anyone can rebuild the app with a modified LAME by editing
   `VinylAlbumRecorder/ThirdParty/lame/` and re-running the Xcode build
   (satisfying the "user can relink" requirement via provided source).

If you ever distribute binaries of a **closed-source** derivative, you would
need to ship LAME as a separate dynamic library (or provide relinkable object
files) instead. Also note the LAME project's own caution that distributing
compiled MP3 encoders may require an MP3 patent license in some
jurisdictions; the last relevant MP3 patents expired worldwide in 2017, so
this is generally considered moot today.

## Encoder configuration used

CBR at 192/256/320 kbps · 44.1 kHz output · joint stereo (mono for mono
inputs) · quality preset 2 · no Xing/Info header (CBR) · ID3 tags written by
the app's own Swift ID3v2.3 writer, not LAME's.
